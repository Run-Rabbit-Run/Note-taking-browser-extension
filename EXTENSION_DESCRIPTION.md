# Rabbit Note: описание расширения

Rabbit Note - браузерное расширение для создания заметок прямо на веб-страницах и видео YouTube. Расширение сохраняет заметки в хранилище браузера, показывает их в popup-окне, позволяет возвращаться к сохраненным местам и экспортировать материалы в Markdown.

## Основные возможности

### Заметки на YouTube

- Добавляет кнопку заметки в панель управления YouTube-плеера.
- Сохраняет закладку на текущем таймкоде видео.
- Автоматически формирует подпись вида `Bookmark at 04:29`.
- Позволяет добавить или изменить текстовую заметку к таймкоду.
- Позволяет запустить видео с сохраненного момента из popup-окна.
- Показывает сохраненные видео-закладки в порядке времени.
- В общем списке заметок открывает видео в новой вкладке и пытается сразу перейти к нужному таймкоду.

### Текстовые заметки на сайтах

- Работает на любых страницах, доступных content script расширения.
- При выделении текста показывает небольшую кнопку рядом с выделением.
- Открывает встроенный редактор заметки поверх страницы.
- Сохраняет выделенный фрагмент, пользовательскую заметку, URL страницы, заголовок страницы и позицию текста.
- Позволяет открыть сохраненную текстовую заметку и прокрутить страницу к исходному фрагменту.
- Для повторяющихся фрагментов текста использует сохраненную позицию, чтобы точнее найти нужное место на странице.
- Позволяет редактировать и удалять заметки текущей страницы из popup-окна.

### Popup-интерфейс

- Показывает заметки для текущей активной вкладки.
- Отдельно отображает видео-закладки и текстовые заметки.
- Поддерживает режим `Show all notes` для просмотра всех сохраненных заметок.
- В общем списке заметки можно группировать:
  - по страницам;
  - по доменам.
- Из общего списка можно открыть сохраненную заметку в новой вкладке.
- Для длинных заголовков используется компонент с tooltip, чтобы интерфейс оставался компактным.

### Экспорт в Markdown

- Экспортирует текстовые заметки текущей страницы в `.md` файл.
- Формирует безопасное имя файла из заголовка страницы.
- Поддерживает стандартный Markdown-шаблон с YAML-блоком:

```md
---
tags:
  - from_site
создал заметку: {{date}}
Link: {{pageUrl}}
---
{{notes}}
```

- Вставляет выделенные цитаты в Markdown как blockquote.
- Добавляет пользовательские комментарии под соответствующими цитатами.
- Поддерживает пользовательский шаблон экспорта.
- Доступные плейсхолдеры шаблона:
  - `{{date}}`;
  - `{{pageUrl}}`;
  - `{{pageTitle}}`;
  - `{{notes}}`.
- Если в шаблоне нет `{{notes}}`, расширение автоматически добавляет заметки в конец файла.

### Пакетный экспорт всех заметок

- Позволяет экспортировать все сохраненные заметки из общего списка.
- Для текстовых заметок использует выбранный Markdown-шаблон.
- Для YouTube-закладок создает Markdown-файлы с тегом `from_video`, ссылкой на видео и ссылками на таймкоды.
- Раскладывает файлы по папкам доменов.
- Обрабатывает одинаковые имена файлов, добавляя числовой суффикс.
- Использует выбранную папку экспорта, если браузер поддерживает File System Access API.

### Настройки

- Открываются через кнопку настроек в popup-окне.
- Позволяют выбрать папку экспорта по умолчанию.
- Сохраняют выбранную папку через IndexedDB и имя папки через `chrome.storage.local`.
- Позволяют сбросить выбранную папку.
- Позволяют выбрать режим Markdown-шаблона:
  - стандартный шаблон;
  - пользовательский шаблон.
- Сохраняют настройки шаблона в `chrome.storage.sync`.

## Архитектура проекта

Проект построен на React, TypeScript и Vite.

### Основные точки входа

- `src/main.tsx` - React-точка входа для popup-окна.
- `src/App.tsx` - основной popup-интерфейс расширения.
- `src/options/main.tsx` - React-точка входа для страницы настроек.
- `src/options/OptionsPage.tsx` - интерфейс настроек экспорта.
- `src/scripts/contentScript.ts` - script, который работает внутри открытых страниц.
- `src/scripts/background.ts` - service worker расширения.

### Компоненты

- `AllNotes` - общий список всех заметок, группировка и пакетный экспорт.
- `TextBookmark` - карточка текстовой заметки.
- `VideoBookmark` - карточка YouTube-закладки.
- `OverflowTooltip` - отображение длинного текста с подсказкой.

### Helper-модули

- `helpers/utils.ts` - активная вкладка, имена файлов, генерация Markdown для текстовых заметок.
- `helpers/exportTemplate.ts` - стандартный шаблон, плейсхолдеры, применение и сохранение шаблонов.
- `helpers/exportDirectory.ts` - выбор папки, сохранение directory handle, запись одного или нескольких Markdown-файлов.
- `helpers/typeUtils.ts` - type guards для разных типов заметок.
- `helpers/createElements/createNoteEditorPopup.ts` - создание встроенного редактора заметки на странице.

## Хранилище данных

Расширение использует несколько механизмов хранения:

- `chrome.storage.sync` - заметки и настройки Markdown-шаблона.
- `chrome.storage.local` - имя выбранной папки экспорта.
- `IndexedDB` - сохраненный `FileSystemDirectoryHandle` для прямой записи Markdown-файлов в выбранную папку.

Для YouTube ключом хранения выступает `videoId`. Для обычных страниц ключом выступает URL страницы.

## Разрешения расширения

В `public/manifest.json` указаны:

- `storage` - сохранение заметок и настроек;
- `tabs` - работа с активными вкладками и открытие сохраненных страниц;
- `downloads` и `downloads.shelf` - экспорт Markdown через стандартный механизм загрузок;
- `host_permissions: <all_urls>` - работа content script на сайтах;
- `content_scripts` для всех URL;
- `background.service_worker` для фоновой обработки открытия заметок и сообщений.

## Поведение экспорта

Если для текущей страницы настроена папка экспорта и браузер выдал разрешение на запись, Markdown сохраняется напрямую в выбранную папку. Если папка не выбрана, API не поддерживается или доступ недоступен, экспорт текущей страницы переходит на стандартный диалог сохранения через `chrome.downloads.download`.

Пакетный экспорт всех заметок требует выбранной папки экспорта, потому что ему нужно создать несколько файлов и папки доменов.

## Сборка и разработка

Доступные команды:

```bash
npm run dev
npm run build
npm run lint
npm run preview
```

Сборка расширения выполняется в папку `dist`. Vite собирает popup, страницу настроек, background service worker и content script отдельными entry point с фиксированными именами файлов.
